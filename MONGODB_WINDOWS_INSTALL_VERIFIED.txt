================================================================================

PHASE 3: MONGODB SERVICE VERIFICATION
================================================================================

□ Step 3a: Verify MongoDB Service Auto-Starts
   
   Method 1 - Check Services (Recommended):
   1. Press: Windows Key + R
   2. Type: services.msc
   3. Press: Enter
   4. Search for: "MongoDB"
   5. Status should show: Running (green arrow icon)
   6. Startup Type should show: Automatic
   
   Method 2 - Check via Command Prompt:
   1. Open Command Prompt (Win+R → cmd → Enter)
   2. Type: sc query MongoDB
   3. Look for line: STATE : 4 RUNNING
   
   Expected output:
   ```
   SERVICE_NAME: MongoDB
           TYPE        : 10  WIN32_OWN_PROCESS
           STATE       : 4  RUNNING
           WIN32_EXIT_CODE : 0  (0x0)
           SERVICE_EXIT_CODE : 0  (0x0)
           CHECKPOINT  : 0x0
           WAIT_HINT   : 0x0
   ```
   
   ✅ If STATE shows "4 RUNNING" → MongoDB is ready!

□ Step 3b: Verify MongoDB is Listening on Port 27017
   
   Method: Check network listening
   1. Open Command Prompt
   2. Type: netstat -ano | findstr :27017
   3. Should show output with 27017
   
   Expected output (example):
   ```
   TCP    127.0.0.1:27017        0.0.0.0:0              LISTENING    1234
   ```
   
   ✅ If you see this → MongoDB is listening and ready!

PROBLEM: "connect ECONNREFUSED" in backend console
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cause: MongoDB service is not running
Solution (choose one):

Option 1 - Start MongoDB service via Command Prompt (QUICK):
  1. Open Command Prompt (as Administrator)
  2. Type: net start "MongoDB"
  3. Press: Enter
  4. Should show: "The MongoDB service has been started"
  5. Go back to backend window (npm start)
  6. Press: Ctrl+C to stop backend
  7. Type: npm start again
  8. Backend should connect now

Option 2 - Start MongoDB via Services GUI:
  1. Press: Windows Key + R
  2. Type: services.msc
  3. Press: Enter
  4. Find: "MongoDB"
  5. Right-click → "Start"
  6. Status changes to "Running"
  7. Close Services window
  8. Restart backend: Ctrl+C then npm start

Option 3 - Restart your computer:
  1. Save all work
  2. Close all terminals
  3. Restart Windows
  4. MongoDB auto-starts on boot
  5. Follow normal startup procedure

Verify fix:
  - Open Command Prompt
  - Type: mongosh
  - Should connect (see "test>" prompt)
  - Type: exit

PROBLEM: "Cannot find module 'mongoose'"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cause: Mongoose package not installed
Solution:

  1. Stop backend: Ctrl+C in backend window
  2. Type: npm install mongoose
  3. Wait for completion (look for "added X packages")
  4. Type: npm start to restart backend
  5. Should connect successfully now

Verify fix:
  1. Type: npm list mongoose
  2. Should show: mongoose@X.X.X

PROBLEM: "MongoDB not found" from mongosh command
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cause: mongosh not in Windows PATH, or MongoDB not installed in standard location
Solution:

Option 1 - Check if MongoDB is installed:
  1. Navigate to: C:\Program Files\MongoDB\Server\
  2. Look for version folder (7.0, 6.0, etc.)
  3. If not found → MongoDB not installed, run installer again

Option 2 - Add MongoDB to PATH:
  1. Press: Windows Key + R
  2. Type: systempropertiesadvanced
  3. Press: Enter
  4. Click: "Environment Variables"
  5. Under "System variables" find: "Path"
  6. Click: Edit
  7. Click: New
  8. Add: C:\Program Files\MongoDB\Server\7.0\bin
     (Replace 7.0 with your actual version)
  9. Click: OK on all windows
  10. Restart Command Prompt
  11. Try: mongosh again

PROBLEM: "Port 5000 already in use"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cause: Another process using port 5000
Solution:

Option 1 - Stop other process:
  1. Find what's using port 5000:
     - Open Command Prompt
     - Type: netstat -ano | findstr :5000
     - Find the PID (number at end)
     - Example: find "12345" at end of line
  2. Kill the process:
     - Type: taskkill /PID 12345 /F
     - Replace 12345 with actual PID
  3. Try npm start again

Option 2 - Use different port:
  1. In backend Command Prompt
  2. Type: $env:PORT=5001; npm start
  3. Backend runs on 5001 instead
  4. Update browser URL: http://localhost:5173 (frontend stays same)

   - Starts automatically with Windows
   - Use services.msc to manage
   - NO .exe file to double-click - it's a service!

3. Command Prompt vs PowerShell:
   - These instructions use Command Prompt
   - PowerShell should work the same
   - Git Bash might have different syntax
   - Recommendation: Use Command Prompt (cmd)

4. File Paths:
   - Use backslash: c:\Users\munee\...
   - NOT forward slash: c:/Users/munee/...
   - Copy/paste paths from File Explorer address bar

5. Port 27017:
   - MongoDB uses port 27017 (standard, don't change)
   - Make sure firewall allows it
   - If port blocked: Check Windows Firewall settings

6. User Account:
   - MongoDB Service runs as Local System Account
   - You don't need admin for normal usage
   - You WILL need admin to install MongoDB

7. Data Files:
   - MongoDB stores data in: C:\data\db\
   - Not affected by your OMEH project folder
   - Persists even if you move OMEH folder
   - Backup this folder to protect data!

8. Mongoose vs MongoDB:
   - MongoDB = Database server (must be installed)
   - Mongoose = Node.js library (npm install)
   - You need BOTH for the project to work
